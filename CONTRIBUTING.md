# Contributing to NoticeUI

Thank you for considering contributing to NoticeUI! We welcome contributions from the community to help make this library better.

## How to Contribute

### Reporting Bugs

If you find a bug, please create a new issue on GitHub and include:

1. A clear description of the issue.
2. Steps to reproduce the behavior.
3. Expected behavior.
4. Screenshots if applicable.
5. Your environment (iOS/macOS version, Xcode version).

### Suggesting Enhancements

Have an idea for a new feature? We'd love to hear it! Please open an issue and clearly explain your idea, why it's useful, and how you imagine it working.

### Pull Requests

1. **Fork the repository** and create your branch from `main`.
2. **Clone the project** to your local machine.
3. **Make your changes**. Ensure you follow the code style (Swift design guidelines).
4. **Test your changes**. Run existing tests and add new ones if necessary. Verify that the Demo app works correctly.
5. **Push to your fork** and submit a Pull Request.
6. **Describe your changes** clearly in the PR description.

## Code Style

- Follow standard Swift API Design Guidelines.
- Ensure strict concurrency checking where possible.
- Use SwiftUI native components.
- Keep the public API clean and minimal.
- Document public methods and properties using DocC-compatible comments.

## Setup for Development

1. Open `Package.swift` or the `Examples/NoticeUIDemo` project in Xcode.
2. Ensure you have the latest stable version of Xcode installed.
